## 1. Project Title

**ElectrifyLabs Homepage** — Static Website from Figma Design

A practice project focused on the full design-to-code pipeline — designed entirely in Figma from scratch, then translated into clean, semantic HTML and CSS with pixel-perfect accuracy.

---

## 2. Description

ElectrifyLabs Homepage is a static website built to sharpen one of the most valuable frontend skills: converting a professional Figma design into real, production-quality code without using any frameworks, libraries, or CSS utilities.

The design was created from scratch in Figma — every component, layout, spacing decision, and color choice was intentional. The development phase then converted that design into clean HTML5 and CSS3, maintaining code quality and semantic structure throughout.

> ⚠️ **Note:** This version is intentionally non-responsive — the focus was on accurate Figma-to-code implementation. Responsive support is planned for a future iteration.

---

## 3. Features

- 🎨 **Original Figma Design** — Designed from scratch by the author; not a copy or template.
- 🧼 **Pure HTML5 + CSS3** — Zero frameworks, zero dependencies, zero shortcuts.
- 🏗️ **Semantic Markup** — Properly structured HTML for accessibility and SEO.
- 📐 **Pixel-Perfect Conversion** — Every spacing, font size, and color matches the Figma source.
- 🧹 **Clean Code Architecture** — Organized, readable CSS with clear naming conventions.
- ⚡ **Performance Optimized** — Lightweight static files with fast load times.
- 🚀 **Live Deployment** — Hosted on Vercel with continuous deployment from GitHub.
- 🔒 **Design-First Approach** — UI/UX decisions made at design time, not during development.

## 5. Getting Started

### Prerequisites

- A modern web browser (Chrome, Firefox, Safari, or Edge)
- A text editor like [VS Code](https://code.visualstudio.com/) *(optional, for editing)*
- No Node.js, npm, or build tools required!

---

## 6. Installation

1. **Clone the repository**

   ```bash
   git clone https://github.com/neelpatel6262/electrifyLabs-home-page.git
   cd electrifyLabs-home-page
   ```

2. **Open the project**

   ```bash
   # Option 1: Open directly in browser
   open index.html

   # Option 2: Use VS Code Live Server
   # Right-click index.html → Open with Live Server
   ```

No build process or dependency installation needed — it's pure HTML and CSS!

---

## 7. Usage

- Open `index.html` in any modern browser to view the site.
- Edit text and images directly in `index.html`.
- Adjust colors, spacing, and typography by editing `style.css`.
- Replace assets in the `Assets/` folder with your own images or icons.
- Push changes to GitHub — Vercel auto-deploys on every commit to `main`.

---

## 8. Keyboard Shortcuts

This is a static marketing page and does not include custom keyboard shortcuts. All standard browser navigation shortcuts apply (`Tab` for focus, `Enter` for links, etc.).

## 10. Development Process

- **Design Phase:** Created a comprehensive UI design in Figma from scratch — all components, layouts, typography, colors, and spacing were planned and finalized before writing a single line of code.
- **Development Phase:** Converted the Figma design to HTML and CSS section by section, maintaining pixel-perfect accuracy and clean code standards throughout.
- **Testing Phase:** Tested cross-browser compatibility across Chrome, Firefox, Safari, and Edge to ensure a consistent rendering experience.
- **Deployment Phase:** Connected the GitHub repository to Vercel for live hosting with automatic deployment on every push.
- **Version Control:** Maintained a clean commit history on GitHub for code tracking and future reference.

## 11. What I Learned

- Designing a complete UI from scratch in **Figma** — components, layout grids, typography, and color systems.
- Converting Figma designs to **pixel-perfect HTML/CSS** without tools like VisBug or browser inspection of existing sites.
- Writing clean, **semantic HTML5** with a proper document hierarchy.
- Creating organized, maintainable **CSS** without relying on frameworks or utility classes.
- Understanding the full **design-to-code pipeline** — from a blank Figma canvas to a live deployed site.
- Setting up **Vercel continuous deployment** from a GitHub repository.
- Practicing **version control workflow** — committing, pushing, and maintaining a clean repo.

## 14. Known Issues / Limitations

- **Non-responsive** — The layout is fixed-width and not optimized for mobile or tablet screens.
- No JavaScript interactions — fully static, no dynamic behavior.
- No accessibility audit completed — ARIA roles and focus states may be incomplete.
- Content is hardcoded in HTML — no CMS or templating engine.
